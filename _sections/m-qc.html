PR-PD-005

Quality Control

Part of the Product Development module — a maturity-style Quality Control assessment in which the Fessociate rates the entrepreneur’s premises against 23 standard QC practices on a 1–5 star scale (Existing), sets a target rating profile (Optimized / Required), and the system auto-computes a per-practice gap, priority and an aggregate QC score (mean of all rated stars) with a maturity tier and rating distribution.
Key behaviour — star-rated maturity scale with an Existing → Required gap engine. The same fixed list of 23 QC practices appears in both the Existing and Optimized tabs. In Existing each practice is rated 1–5 stars on what is observed at the entrepreneur’s premises; in Optimized the target star level is set. For every practice rated in both tabs, the system calculates Gap = Target − Existing and derives a Priority (Critical ≥ 3, High ≥ 2, Low = 1, At Target = 0). The aggregate QC score per tab is the mean of all rated stars, rounded to a maturity tier. A single “Intervention Need” toggle (with justification) on the Existing tab flags the whole assessment for improvement; the Optimized tab then carries one Implementation Status.

On-Screen UI Structure

Top bar: teal header “Quality Control” with crumb “PRIME Rural · Quality & Compliance”, and an entrepreneur chip (avatar + name) auto-fetched from the profile.
Tab rail: horizontal pill bar with five tabs — Summary · Existing · Optimized · Resources · Task Log; the active tab renders teal-on-white (Summary opens by default).
Body layout: each tab is a panel. Summary is read-only: a Practice-wise Overview table (one row per practice, Existing vs Target with gap + priority), a 2×2 grid of four Aggregate cards (Existing Assessment / Required Target / Interventions / Rating Distribution) and a 4-cell Intervention-flag strip. Existing and Optimized each show a 5-tier star Legend strip followed by a 23-row Assessment table and an aggregate Average strip.
Special widgets: (1) Star-rating control — each of the 23 assessment rows carries a 5-star clickable rating (hover preview, click to set, × clear button) plus a free-text Notes field. (2) Legend strip — a 5-cell key mapping 1–5 stars to maturity labels (Not aware / Not practiced → Fully implemented). (3) Average strip — a teal gradient panel showing Practices Rated (n / 23), the numeric Average Rating and an average-star rendering. (4) Intervention Need toggle (Existing) — an on/off switch “Need improvement in Quality Control?” that reveals a conditional Justification textarea. (5) Implementation Status selector + Notes (Optimized). (6) Distribution bars — a 5-tier bar chart of how many practices fall in each star band.
Per Rule 1, every calculated / autofilled field (all Summary aggregates and overview cells, both Average strips, the per-practice Gap and Priority, the maturity tier, the distribution counts) renders italic on a cream-deep background and is non-editable, with a tooltip describing its source / formula.

Entirely auto-populated from the Existing and Optimized tabs — no data entry on this tab. Rows with a gap ≥ 2 render on an amber background (row-gap2); gap ≥ 3 render orange (row-gap3).

Key validations (whole form): The aggregate QC score on each tab is the mean of rated stars only (sum ÷ rated count) — unrated practices are excluded, not counted as zero. Existing and Optimized share the same fixed 23-practice list; a per-practice Gap (Target − Existing) and Priority are computed only when a practice is rated in both tabs, otherwise Gap / Priority show “—”. Average Gap floors negative gaps at 0. Maturity Tier is keyed by round(average) onto the five fixed labels. The Existing “Need improvement?” toggle, when on, reveals (and expects) a Justification and sets Summary “Intervention Flagged” = Yes. Task Log rows auto-renumber on add / delete and seed Date to today.
