PR-SK-003

Business Plan Canvas

Skilling
Entrepreneur-level framework. A one-page strategic synthesis of the entrepreneur's business model across 9 building blocks, grouped into four quadrants: Infrastructure, Offering, Customer, and Financial. Each block surfaces a cross-module reference so the Fellow can draw from operational data already captured in other diagnostic frameworks. The canvas is filled collaboratively by the Fessociate with the entrepreneur and reviewed by CTIT.
DocType: DF Business Plan Canvas — single entrepreneur-level doctype, no child tables. Nine textarea fields (one per block) + a Task Log child table sourced from Framework Task Master (BPC sheet, 17 tasks). Resources child table for reference material uploads. Fields: key_partners, key_activities, key_resources, value_propositions, customer_relationships, channels, customer_segments, cost_structure, revenue_streams.
Cross-module references (read, not auto-populated): Key Partners → Market Linkage • Key Activities → Standardisation • Key Resources → HR + Infrastructure (Machinery) • Channels → Promotional Marketing • Customer Segments → Customer Analysis • Cost Structure → Unit Pricing • Revenue Streams → Financials. The Fellow reads these modules first and synthesises the insight into the canvas block manually — no field values are copied automatically.

User Journey ▼
  1. Fessociate opens entrepreneur’s Skilling module → selects Business Plan Canvas framework.
  2. Summary tab loads showing all 9 blocks as Empty with 0% completion.
  3. Fessociate navigates to Business Plan Canvas tab — sees the 9-block canvas grid.
  4. For each block, Fessociate clicks the reference pill (e.g. “Ref: Market Linkage”) to review operational data already captured in that module, then returns and types the synthesised insight into the textarea.
  5. Summary tab updates live — block status flips to Filled, completion percentage rises, quadrant counts update.
  6. Fessociate logs activity in Task Log tab, selecting task from dropdown (e.g. “03 — Fill Key Partners Block”).
  7. After all 9 blocks filled, Fessociate uses Resources tab to attach any supporting templates or reference files.
  8. CTIT reviews the completed canvas, checks Intervention Flags (especially Value Prop Set and Financial Blocks), and approves or requests revisions.
  9. Activity syncs to Fellow Activity Logger Log Book for this framework and to Entrepreneur Profile Intervention Logs (read-only).
Key Validations ▼
RuleDetail
No auto-populateAll 9 canvas blocks are manually filled. No field values are copied from cross-module references — the Fellow synthesises the insight.
Canvas blocks are free textNo character limit enforced per block, but UI shows a guiding placeholder. All 9 blocks are optional on save; at least the Value Proposition block is flagged in Intervention Flags if empty.
Task dropdown filters to BPCFramework Task Master dropdown is filtered to framework = “Business Plan Canvas”. Phase column hidden from Fellow view; visible to CTIT only.
“Other” task handlingWhen Fellow selects “If other, specify in comments”, the Notes / Outcome column captures the free-form description — no additional field added.
Summary is read-onlySummary tab values are computed from canvas textarea content; they are never directly editable by the user.
CTIT-only reviewFinal canvas review and approval recorded against CTIT role. Fessociate fills; CTIT reviews and may request revisions via BRD review panel.
