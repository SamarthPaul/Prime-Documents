PR-INF-001

Infrastructure

A single Infrastructure framework with three intervention tabs — Built Environment (BE), Energy Efficient Machinery (EE) and Clean Energy (CE) — plus Summary, Financial Model, Tracker, Task Log and Resources. The screen opens on a dashboard-style Summary, then drills into the per-tab forms. The SELCO Executive works inside the same framework but edits only SELCO-relevant fields (pricing, equipment status, disbursement); funding is split across SELCO / MBMA / LIFCOM / USER with a LIFCOM EMI option.
Key behaviour: Infrastructure starts from a dashboard overview, then drills into forms. SELCO Executive sees all tabs but edits only SELCO-relevant fields (pricing, equipment status, disbursement). Required machines arrive from the Machinery framework via dropdown selection.

On-Screen UI Structure

Top bar: forest-green header reading “Infrastructure” with a breadcrumb “PRIME Rural · Project Setup”, and an entrepreneur chip (avatar + name) auto-fetched from the EP profile.
Tab rail: a horizontal pill rail — Summary · Built Environment · Energy Efficient Machinery (Machines) · Clean Energy (Solar) · Financial Model · Tracker · Task Log · Resources. Only one panel is visible at a time; switching to Summary / Financial Model / Tracker triggers a live recompute.
Body layout: each tab is a stack of numbered Roman-numeral section blocks (I, II, III…) containing cream sub-sections of label + control field rows. Implementation Status blocks are terracotta-themed with a single status banner + a child “log” table.
Special widgets: Summary uses a component-wise overview table (one row per BE/EE/CE), a 2×2 aggregates grid (Infrastructure Costs, Funding Split, Interventions, Disbursement & Tracker) and an intervention-flag strip. The Financial Model has a tier/slab A/B/C funding-split reference (SELCO/MBMA/LIFCOM/USER %), four left-border funder cards that switch between static % (PRIME) and editable % inputs (Innovation/Scale), a % validation strip, and a LIFCOM EMI calculator (5-year / 3-year cards). The Tracker shows allocated-vs-disbursed tranche cards (cascading MBMA → USER → SELCO → LIFCOM), a vendor-payment disbursement table and a Disbursed − Paid balance card. EE machines render as collapsible cards with an auto-fetched detail panel.
Calculated fields (Plinth Rate, Rent/Unit, Electricity/Unit, BE Estimated Cost, EE/CE totals, all funder amounts, EMI, tranche allocations, balance) render italic on a cream-deep / info background with a formula tooltip per Rule 1.

Key validations (whole form): (1) BE/CE intervention “Justify” is required whenever the corresponding intervention toggle is On. (2) MSOR typologies auto-calc BE cost (Plinth Rate × SQM) and lock the cost field; non-MSOR typologies (Kiosks) require a manual cost entry. (3) SQM and SQF must stay in sync via factor 10.7639. (4) Innovation/Scale funder percentages must sum to exactly 100% (SELCO + MBMA + LIFCOM + USER + Others). (5) PRIME slab is cost-driven; Total ≥ ₹25,00,000 is “Out of policy” and needs manual override. (6) EMI (PRIME only) = LIFCOM amount ÷ 60 (5yr) or ÷ 36 (3yr) — flat, no interest. (7) Tranche disbursement must not over-allocate a cost head; uncovered amounts raise a funding-gap warning. (8) Balance with Entrepreneur = Total Disbursed − Total Paid. (9) SELCO Executive may edit only SELCO-relevant fields (SELCO price, equipment status, SELCO approval, solar capacity/cost, implementation status banners, Final cost totals, disbursement & vendor payments) — all other fields are read-only to that role. (10) Required machines must be selected from the Machinery framework dropdown; their detail panel is read-only here.
