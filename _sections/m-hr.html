PR-MFG-003

Human Resource

Captures the enterprise workforce (categories, skill levels, wages and headcount) under the Manufacturing module's diagnostic framework. Each worker category's monthly labour cost and labour-rate-per-hour feed Unit Pricing individually, while every wage line is checked against Government of Meghalaya (GoM) minimum-wage thresholds for compliance flagging.
Key behaviour — Existing → Required intervention bridge: When the Fessociate toggles "Labour intervention needed?" on an Existing worker block, a linked row is auto-created in the Required tab (Section I, Labour Interventions). The bridge is two-way: the Existing worker's Category / Skill / Nature / Justification flow into the Required row as read-only context, and removing the Required row (or unflagging the toggle) clears the bridge on both sides. New Labour (Required Section II) is standalone and is NOT bridged to Existing.

On-Screen UI Structure

Top bar: teal header "Human Resource Module" with breadcrumb "PRIME Rural · DF Human Resource". Below it an entrepreneur chip (avatar + name) auto-fetched from the entrepreneur profile.
Tab rail: a horizontal rail of five tabs — Summary · Existing · Required · Resources · Task Log. Active tab is teal-filled; switching to Summary triggers a full recalculation.
Body layout: each tab is a panel. Existing renders one collapsible worker block (child table row) per category, each with three sub-sections (Worker Classification, Wage Structure, Cost & Compliance) plus an Intervention-Need toggle. Required holds two sections — I. Labour Interventions (bridged rows) and II. New Labour (standalone blocks). Summary shows a worker-wise overview grid, four aggregate cards, and four intervention-flag tiles.
Special widgets: a per-block GoM compliance flag (green "Yes" / red "No" condition-badge) and a red wage-warning banner when the implied daily wage falls below the GoM minimum for the skill level; collapsible block headers carry Skill and GoM✓/GoM✗ tags.
Calculated fields (Total Workers, Monthly Labour Cost, Labour Rate / hr, GoM Compliant?, Required Monthly Cost) are rendered italic on a cream-deep / highlight background, read-only, with the active formula shown in the "Cost Formula Applied" strip (Rule 1).

Key validations (whole form): (1) Worker Category, Skill Level, Employment Nature, Wage Type and Wage Rate are mandatory on every Existing and New Labour block. (2) Male / Female / Days / Hours / Wage Rate must be ≥ 0 — no negative values. (3) Days is required for Per Day & Per Hour cost formulas; Hours/Day is required for Per Hour. (4) GoM compliance is enforced as a flag, not a hard block: any block whose implied daily wage < the skill-level GoM minimum (Unskilled ₹360 / Semi-skilled ₹410 / Skilled ₹480 / Highly Skilled ₹550) is marked Non-Compliant with a red warning banner and counts in the GoM Non-Compliant tile. (5) A flagged Existing block should carry a justification; the bridged Required row then requires an Intervention Type. (6) Only Status = Completed Required rows propagate to Summary “After” cost, Workers After and Jobs Created. (7) Each worker category's Monthly Labour Cost feeds Unit Pricing individually — costs are never pre-summed before being passed downstream.
