PR-MFG-002

Machinery

Captures every machine and tool the entrepreneur owns or needs, computes per-machine depreciation that flows individually into Unit Pricing, and is part of the Manufacturing module. Golden-Thread ties: machines named in a Process Flow pre-create rows in the Existing tab, and machines flagged as Required surface in Infrastructure / EE workflows through explicit dropdown selection (never auto-pushed).
Key behaviour — the Existing → Required intervention bridge. A machine is recorded ONCE in the Existing tab. Toggling its “Servicing Needed?” or “Capacity Upgrade Needed?” flag does not capture any cost — it only raises a flag. The Fessociate then opens the Required tab and adds an Intervention Row, picks the flagged machine from a dropdown of Existing machines, and the flag type + reference (current cost · condition) auto-fetch. New (additional) machines are recorded separately in Required → Section II with the full Existing-style structure plus a Procurement Status. Each machine’s Depreciation / Unit feeds Unit Pricing individually — the engine reads every machine row, not a single total.

On-Screen UI Structure

Top bar: teal header “Machinery Module” with crumb “PRIME Rural · DF Machinery”, and an entrepreneur chip (avatar + name) auto-fetched from the profile.
Tab rail: horizontal pill bar with five tabs — Summary · Existing · Required · Resources · Task Log; active tab renders teal-on-white.
Body layout: each tab is a panel. Summary is read-only: a machine-wise overview table, four aggregate cards (Machines / Capacity / Cost / Interventions & Status), and a 4-cell flag strip. Existing and Required render each machine as a collapsible “machine block” (sea-blue header with name input + Remove) holding four sub-sections: Classification & Process Integration, Technical & Power Specifications, Operational Capacity & Usage, Financial & Maintenance Data.
Special widgets: (1) Depreciation / capacity calc strips — highlighted cells inside each block showing Monthly Output, Capacity Utilization, Capacity Gap, Depreciation Rate, Monthly Depreciation and Depreciation / Unit, each derived live. (2) Intervention banners — in Existing, a terracotta “Intervention Need — Flag Only” panel with two toggles; in Required, collapsible intervention rows with Machine / Flag / Implementation Status selectors in the header. (3) A teal total bar at the foot of Existing and Required summarising counts, investment, output and total depreciation, and naming the Unit Pricing target.
Per Rule 1, every calculated field (Monthly Output, Utilization, Gap, Depreciation Rate, Monthly Depreciation, Depreciation / Unit, all totals and aggregates) renders italic on a cream-deep background and is non-editable, with a tooltip stating its formula.

Tab 1 — Summary (read-only roll-up)

Entirely auto-populated from the Existing and Required tabs. No data entry on this tab. It contains four blocks: (1) a Machine-wise Overview table, (2) four Aggregate cards, (3) an Intervention Flag strip. Each machine’s Depreciation / Unit feeds Unit Pricing individually; the “TOTAL (indicative)” row is informational only because Unit Pricing reads each machine separately.

Tab 3 — Required (Section I: Existing Machine Interventions · Section II: New Machines)
Bridge from Existing (Rule 3). Section I rows do not re-enter machine data — they link to a machine already in the Existing tab via a dropdown built from Existing machine names. On selection, the machine’s active flag (Servicing / Capacity) and a Reference (current cost · condition) auto-fetch. Section II New machines are fully described here; if a New machine is later marked Required for the EE, it surfaces in Infrastructure / EE through explicit dropdown selection there — it is never auto-pushed.

Key validations (whole form): Name of Machinery / Tool and No. of Machinery / Tool are mandatory on every Existing and New machine block (quantity ≥ 1). Days (0–31), Hours (0–24), all capacity, cost, wattage, upkeep and age values must be ≥ 0; Current Usage Capacity must not exceed Max Processing Capacity. Machinery Condition must be set for depreciation to compute (otherwise depreciation shows “—”). “If Critical, provide reason” is required when the machine is marked Critical. In Required → Section I, every Intervention Row must reference a machine that exists in the Existing tab (dropdown selection); intervention cost and post-solar electricity cost must be ≥ 0. Resources require at least one row. Depreciation / Unit feeds Unit Pricing per machine — never as a single aggregate. Required machines reach Infrastructure / EE only by explicit dropdown selection, not auto-push.
