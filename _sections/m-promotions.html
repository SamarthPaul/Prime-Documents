PR-MKT-004

Promotions

Part of the Market module, the Promotions framework maps how an entrepreneur currently promotes their products across all offline and online (digital) channels, flags whether new promotion methods are needed, proposes recommended channels with an implementation status, and tracks promotional / digital-marketing activities through a task log and resource library. Backed by DF Promotional Marketing (one record per entrepreneur) with Promotional Marketing Entry child rows phased Existing / Required.

On-Screen UI Structure

Top bar: forest-teal header "Promotions" with a "PRIME Rural · Marketing Framework" crumb, and an entrepreneur chip below it (avatar + name auto-fetched from the Entrepreneur Profile).
Tab rail (uppercase pills): Summary · Existing · Required · Resources · Task Log. Active tab is teal-filled; the form opens on the Summary tab (which refreshes on open).
Body: the Existing and Required tabs are channel repeaters ("+ Add Promotion" appends a row) backed by the same Promotional Marketing Entry child table, differentiated by a Phase field (Existing vs Required). The Existing tab additionally carries an Intervention Need block (toggle + justification). The Required tab adds an Implementation Status column. Resources and Task Log are independent child-table registers.
Special widgets: (1) an Intervention Need toggle (forest-green when on) that reveals a justification textarea and flags the record for new promotion methods; (2) a conditional Digital Platform Category column that becomes relevant only when Channel Type = Online (NA when Offline); (3) an Implementation Status pill (Pending / In Progress / Completed / Not Feasible) on Required rows; (4) a Summary tab with a Promotion-wise Overview table, four Aggregate cards (Existing / Cost-Frequency / Interventions / Intervention Status) and four Intervention-flag tiles; (5) Upload / View file actions on Resources and Task Log rows.
Calculated fields (entrepreneur chip, all Summary overview rows, aggregate counters and flag tiles, the live Task Log entry counter) render italic on cream-deep background, read-only, with a tooltip per Rule 1.

Key validations (whole form): (1) Entrepreneur is required and unique — one DF Promotional Marketing record per entrepreneur; duplicate rejected. (2) Module ("Marketing") and Applicable To ("Product") are read-only system defaults; District & Block are read-only fetches from the Entrepreneur Profile and must never be hand-edited. (3) Every promotion row needs a Promotion Name to count in the Summary — "Enter the promotion name." (4) Phase is system-set (Existing on the Existing tab, Required on the Required tab). (5) Implementation Status is mandatory on Required rows ("Set the implementation status"); blank is treated as Pending. (6) Digital Platform Category is conditional — relevant when Channel Type = Online, NA when Offline. (7) When the Intervention toggle is On, a Justification is expected and the record is flagged "New Methods Needed = Yes". (8) Resources table must retain at least one row ("At least one row is required."). (9) Task Log entries require a Task Name ("Select a task."). (10) All Summary figures are system-calculated, read-only, and refresh on open; counts use Indian number formatting.
