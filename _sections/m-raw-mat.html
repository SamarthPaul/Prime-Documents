PR-MFG-001

Raw Materials

Captures every raw material a manufacturing entrepreneur uses — its identity, supply/procurement profile, batch-level cost build-up and the resulting Raw Material Cost / Unit — and records the intervention work (alternate sources, alternate materials, new materials) needed to optimise it. This is the first framework in the Manufacturing module.
Golden Thread. Product names flow in from the entrepreneur's Product Profile (read-only per Rule 2). Each material's calculated RM Cost / Unit and Total Landed Cost / Unit feed Unit Pricing individually — Existing rows feed Existing Unit Pricing, Required (optimised) rows feed Optimised Unit Pricing. Toggling an intervention flag in the Existing tab auto-creates a linked row in the Required tab (intervention bridge, Rule 3); removing the Required row clears the flag and vice-versa.

On-Screen UI Structure

A teal title bar reads Raw Material Module with crumb PRIME Rural · DF Raw Material. Directly below sits a horizontal tab rail — Summary · Existing · Required · Resources · Task Log — the active tab highlighted teal. An entrepreneur chip ("Auto-fetched from profile") sits at the top of every panel.

Summary is a read-only roll-up: a material-wise overview table (one row per material, two right-most columns shaded terracotta/green for New Price & Saving), four aggregate cards (Materials / Cost / Interventions / Status) and a four-tile Intervention Flags strip.

Existing renders each material as a collapsible block (sea-green header carrying inline Product + Raw-Material name inputs and auto Middleman / Essential tags) holding four sub-sections: Material Identity, Supply & Procurement, Cost & Calculations Inputs, and a Calculated strip plus a monospace Calculation Summary. A terracotta Intervention Need banner with two toggle cells closes each block. A teal total bar sums all blocks.

Required has three numbered sections (I Alternate Sources, II Alternate Materials, III New Raw Material). Sections I & II are populated by the intervention bridge — pre-created rows show the originating Product / Raw Material / Justification as read-only (italic on cream-deep), then editable optimisation fields and a terracotta Implementation Status block. Section III rows are added freely.

Calculated fields (Amount Per Batch, RM Qty / Unit, RM Cost / Unit, Transport / Unit, Total Landed Cost / Unit, Expected Saving / Unit) render italic on cream-deep background and are never editable; each carries a formula tooltip (Rule 1).

Intervention bridge (Rule 3). Key validations (whole form): Product Name, Name of Raw Material, Current Status of Use, Is It Essential?, Unit, Quantity Required Per Batch, Price Per Unit and Batch Size are mandatory on every Existing material. Batch Size must be > 0 — all per-unit calculations (RM Qty/Unit, RM Cost/Unit, Transport/Unit, Total Landed Cost/Unit) return 0 when Batch Size is 0 or blank. All currency / quantity inputs must be ≥ 0. An intervention flag toggled ON must have a Justification before save, and toggling it ON/OFF must keep the linked Required row in sync (create on ON, delete on OFF; deleting the Required row resets the flag). Summary “After” and “Saving / Unit” count Completed interventions only. New-material rows use “On Hold” as the fourth status (not “Not Feasible”).
