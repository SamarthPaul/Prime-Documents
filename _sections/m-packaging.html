PR-PD-006

Product Packaging

Product Development framework (DocType DF Packaging) that records, per product variant, every existing packaging item — type, material, source, dimensions, capacity, price, print-capability and sealing — together with a months-as-rows monthly order/quantity grid that aggregates spend across the cohort year. Each variant’s annual packaging cost feeds the Packaging line of Unit Pricing, and an intervention bridge carries flagged items into a Required tab for replacement / new packaging.
Golden Thread. Variants are selected from a dropdown sourced from the entrepreneur’s Product Profile — they are not auto-prepopulated (the framework records only the variants the entrepreneur actually packages). The shared Monthly Quantity Ordered grid spans cohort years (Rule 11): months are rows, each variant’s packaging type is a column, and Annual Cost = Σ(month qty × price/unit) auto-fetches to Existing Unit Pricing → Packaging. Toggling Packaging Intervention Needed? in the Existing tab auto-creates a linked row in the Required tab (intervention bridge, Rule 3); removing the Required row clears the flag, and vice-versa.

On-Screen UI Structure

A teal title bar reads Packaging Module with crumb PRIME Rural · DF Packaging. Below it sits a horizontal tab rail — Summary · Existing · Required · Requirement Tracker · Resources · Task Log · Log Book — the active tab highlighted teal. An entrepreneur chip (“Auto-fetched from profile”) sits at the top of every panel; Entrepreneur, District, Block, Module (Product Development) and Applicable To (Product) all autofill from the Enterprenur Profile (Rule 2).

Summary is read-only: a product-wise overview table (one row per packaging type per variant, the two right-most columns shaded green for New Price & Saving), four aggregate cards (Existing / Cost / Interventions / Implementation Status) and a four-tile Intervention Flags strip.

Existing renders each variant as a collapsible block (sea-green header carrying an inline variant-selector dropdown + SKU). Each block holds Section I — a repeating packaging detail table (one row per packaging item: type, material, source, L×B×H dimensions with unit, capacity + unit, price/unit, supports-printing, picture, and a multi-select sealing pill group) — and Section III, a terracotta Intervention Need banner with packaging + sealing toggle/justify pairs per type. Below all blocks sits the shared Section II Monthly Quantity Ordered grid (months ↓ rows, variant packaging types → columns) and a teal total bar.

Required has two numbered sections: Section I Packaging Interventions (bridge-populated rows showing the originating Product / SKU / Current Type / Material / Price / Annual Qty as read-only, then an editable Required Packaging detail row + sealing + Implementation Status) and Section II New Packaging (standalone items added freely). Requirement Tracker mirrors Section II for the required side (Monthly Quantity Required grid). Resources, Task Log (14 seeded activities) and Log Book close the rail.

Calculated fields (Month Total ₹, Annual qty per type, Annual Cost, Avg Monthly Qty, Saving / Unit) render italic on cream-deep background, are never editable, and each carries a formula tooltip (Rule 1).

Key validations (whole form): Entrepreneur is required and unique (one DF Packaging per entrepreneur). Each Existing block must have a Product Variant selected from the dropdown (Product Profile sourced, not auto-prepopulated) and the same variant cannot be picked twice. At least one packaging detail row per variant; Type of Packaging is mandatory on every row (Existing & Required). Quantity Ordered, Capacity and Price Per Unit must be ≥ 0; Order Year is required on monthly entries (the grid aggregates across cohort years, Rule 11). Calculated fields (Month Total, Annual qty/type, Annual Cost, Avg Monthly Qty, Saving) are read-only with formula tooltips (Rule 1). The Annual Cost auto-fetches to Existing Unit Pricing → Packaging — do not edit it there.
