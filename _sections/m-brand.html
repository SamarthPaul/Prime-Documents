PR-MT-001

Branding Identity

The Branding Identity framework under the Marketing Tools module captures an enterprise's brand story, USP and tagline, then drives three implementation tracks — Brand, Logo and Business Card — each with a Designer-led draft cycle and dual approval from the entrepreneur and the core team. A Summary view rolls up approval status across all three tracks, and a Task Log records the 24-step branding journey.
Key behaviour: the Designer (or AI Generated option) uploads up to three sequential draft versions per track; each track advances only when both the Entrepreneur Approval and Core Team Approval reach Approved. Either party can return a track by marking Rejected, which re-opens the next draft slot.

On-Screen UI Structure

Top bar: teal app header reading “PRIME Rural | Marketing Tools” with a breadcrumb “Diagnostics / Marketing Tools” on the right.
Tab rail: a horizontally scrollable uppercase tab strip — Summary (default active), Brand, Logo, Business Card, Resources, Task Log. The active tab is filled teal.
Summary body: a brand profile card (teal hero) showing a logo upload placeholder, Enterprise label with brand name, italic tagline, and a right-aligned Market Scope value; below it a cream USP strip and a two-column row showing Current Brand Name and New Brand Name. Beneath sits a 3-column grid of track status cards (Brand / Logo / Business Card), each surfacing approval status chips. All Summary values are derived read-outs of data entered on the working tabs.
Brand / Logo / Business Card bodies: grouped into banner-led blocks — teal banners denote assessment/positioning data, terracotta banners denote implementation/action (Designer, Drafts, Approvals). Inside, label-and-value field rows are used, with auto-fetched fields shown italic on a cream-deep background per Rule 1.
Special widgets: a market-scope chip group (single-select pills), Yes/No intervention toggles, Upload / View buttons for HD assets and QR codes, a sequential Draft Versions list with status dots (active vs. pending), and Entrepreneur / Core Team approval status dropdowns. The Logo track Designer dropdown additionally offers an AI Generated option.

Key validations (whole framework): (1) Each implementation track (Brand, Logo, Business Card) is finalised only when both Entrepreneur Approval and Core Team Approval are Approved — a single Rejected blocks delivery and re-opens the next draft slot. (2) A Designer must be assigned before any draft can be uploaded; only the assigned Designer (or core team) holds upload permission. (3) Draft Versions are sequential and capped at 3 (Draft 2/3 stay “Pending” until the prior draft exists/is rejected). (4) “Need New Brand Name?” On → New Brand Name + Justification required; “Need New Logo?” On → Justification required. (5) Market Scope must always have exactly one chip selected. (6) Task Log rows require a Task Name selection; serial No. is system-generated. (7) Resources / Task Log child tables must retain at least one row.
