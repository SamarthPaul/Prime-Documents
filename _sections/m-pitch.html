PR-FN-001

Pitch Deck

Entrepreneur-level framework (Funding module) that guides the entrepreneur through building an 11-slide investor pitch deck. The Existing tab records any deck already held and flags whether help is needed; the Required tab is a structured slide-by-slide content workspace plus the finished-deck upload and implementation status.
Existing → Required bridge (Rule 3). Toggling Does Entrepreneur need a Pitch Deck? ON in Existing flags the intervention and activates the Required tab’s slide workspace. The eleven slides are: Vision & Impact · Problem Statement · Solution (USP) · Business Model · Financial Projections · Milestones · Competition · Marketing Strategy · Team Overview · Expectations from Incubation · Ask (Funding). Slide-fill progress and the final-deck upload feed the Summary aggregates.

On-Screen UI Structure

A teal title bar reads Pitch Deck; a horizontal tab rail — Summary · Existing · Required · Resources · Task Log · Log Book — sits below, active tab teal, entrepreneur chip on every panel (Rule 2).

Existing has two cards: Section I Pitch Deck (upload existing deck .pdf/.ppt/.pptx with a View button, Pitch Video Link with Open ↗, Last Updated date) and a terracotta Section II Intervention Need (Need-a-deck toggle + Justify). Required has Section I Assessment — an 11-slide content grid (each slide is a colour-coded card with title, guiding description, and a content textarea) — and a terracotta Section II Implementation Status (status select, final-deck upload, video link, last updated). Summary is read-only: a slide-wise overview table, aggregate cards (Existing Assets / Slide Progress / Interventions / Slide Breakdown) and four intervention-flag tiles (Slides Filled, Intervention Needed, Required Deck Uploaded, Tasks Completed). Resources, Task Log and Log Book close the rail.

Key validations (whole form): Entrepreneur is required and unique (one Pitch Deck form per entrepreneur). Video links must be valid URLs; deck uploads accept .pdf / .ppt / .pptx only. Slides Filled, Slides Empty and Completion % are computed read-only from the 11-slide grid (Rule 1). The Existing need-a-deck toggle activates the Required slide workspace; the Required Implementation Status and final-deck upload drive the Summary flags. Slide content and the Ask feed the entrepreneur’s funding narrative (links conceptually to Scheme Funding).
