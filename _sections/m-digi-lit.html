PR-SK-001

Digital Literacy

Part of the Skilling (Upskilling) module. It captures the entrepreneur's current digital footprint — device/internet access plus a star-rated proficiency assessment across nine platform skills — flags where training is needed, then carries those flags into a Required tab where target skill levels and implementation status are tracked through delivery, with the Summary rolling up the existing-vs-target skill change and intervention progress.
Entrepreneur-scoped, one record per entrepreneur. The header chip is auto-fetched from the Enterprenur Profile (Entrepreneur is a unique Link; District & Block are hidden fetched fields). Module is fixed to Upskilling and Applicable To is fixed to Enterpreneur (both read-only). The Existing and Required skill grids are backed by the Digital Literacy Entry child table (phase = Existing / Required); flagging a row for intervention in the Existing tab auto-bridges a matching row into the Required tab.

On-Screen UI Structure

A teal title bar reads Digital Literacy with crumb PRIME Rural · Capacity Building. Below it sits a horizontal tab rail — Summary · Existing · Required · Resources · Task Log · Log Book — the active tab highlighted teal. An entrepreneur chip (avatar + name, auto-fetched from profile) sits above every tab body.

The Existing tab has two Roman-numbered sections: I Access & Devices (a checkbox group — Smartphone, Laptop/Computer, Internet) and II Skill Assessment, a special skills grid with one row per platform skill. Each grid row carries a 5-star proficiency rating widget (click to set, × to clear), an Intervention Required? toggle, and a Justification text input. A 5-cell legend above the grid maps stars to proficiency labels. Below the grid an average strip (teal gradient) shows Skills Rated, Average Skill Level and an average-stars readout; a teal total bar repeats the totals and notes the auto-fetch to the Required tab.

The Required tab mirrors this: I Access & Device Needs (three device cards — Smartphone, Laptop/Computer, Internet — each with a Needed checkbox + a status dropdown) and II Target Skill Assessment, the same skills grid but with a Target Skill Level star rating, an Implementation Status dropdown and a Learnings/Notes input. Rows bridged from Existing arrive pre-populated with a ↻ Bridged pill and a platform locked read-only. A status-overview total bar tallies Pending / In Progress / Completed.

The Summary tab is fully read-only: a skill-wise overview table (Existing Level · Intervention Flagged · Target Level · Skill Change · Implementation Status), four aggregate cards (Existing / Required / Skill Change / Implementation Status) and a four-tile intervention-flags strip. Resources is a tile list of training guides; Task Log is a fixed activity checklist; Log Book is an add-row table of training/visit entries.

Calculated fields (Skills Rated, Average / Target Average Skill Level, average-stars, all Summary counts & Skill Change) render italic on cream-deep background, are never editable, and carry a formula tooltip (Rule 1). The entrepreneur chip, District and Block are fetched and read-only with a source note (Rule 2).

Key validations (whole form): Entrepreneur is a unique Link — one Digital Literacy record per entrepreneur; District & Block are fetched read-only; Module (Upskilling) and Applicable To (Enterpreneur) are read-only. In the skill grids a row counts only once a Platform / Skill is selected; a star rating of 0 (unrated) is excluded from Skills Rated and the average. Turning on an Existing row's Intervention toggle auto-bridges a Required row (platform locked, justification synced); deleting either side keeps the two in sync. Devices marked Needed in Required but absent in Existing surface as a Device Gap. Required Implementation Status defaults to Pending and drives the status totals. All Summary figures are read-only and recalculated on every change; numeric roll-ups use Indian number formatting.
