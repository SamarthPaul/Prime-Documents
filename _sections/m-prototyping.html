PR-PD-002

Product Prototyping

Part of the Product Development module, this framework lets the Fessociate track every prototyping iteration for each of an entrepreneur's products — covering both packaging prototypes and product prototypes — from first concept through to scale implementation. One product card is added per product, and each prototype version (v1, v2, …) is logged inside that product's child table with a design upload and key learnings.

On-Screen UI Structure

The screen header reads Product Prototyping with breadcrumb PRIME Rural · Product Development. A horizontal tab rail sits directly below the header with four tabs: Summary, Prototypes, Resources and Task Log.

The Prototypes tab is the data-entry body. It opens with an auto-fetched entrepreneur chip (read-only, pulled from the profile) and a guidance caption. Below sits a repeatable list of collapsible product blocks, each added via a + Add Product button. Every product block has a coloured header carrying three controls: a Product dropdown, an Implementation Status dropdown, and a status chip that mirrors the chosen status. The block body holds two subsections: Prototype Configuration and a Prototype Iterations child table.

A conditional dropdown drives the configuration: Type of Prototyping (Packaging / Product) changes the prompt and expected values of the Prototyping Type Name field below it — Packaging guides the user toward the Packing Master list, Product toward the Business Basket list.

The Summary tab is fully read-only and auto-calculated from the Prototypes tab: a product-wise overview table, four aggregate cards (Products, Iterations, Progress, Intervention Status) and four intervention-flag tiles. The Resources and Task Log tabs each hold a single flat child table for reference materials and dated activity entries respectively.

Key validations (whole form): • Each product block requires a Product before it counts toward Summary — “Select a product.” • Once Type of Prototyping is chosen, Prototyping Type Name becomes mandatory and must belong to the correct master (Packing Master for Packaging, Business Basket for Product); changing the type re-validates and clears a non-matching name. • Design uploads are restricted to .pdf / .jpg / .jpeg / .png / .svg / .ai / .psd. • The Resources table must always retain at least one row. • Task Log entries require a Task Name, and the “If other” option requires Notes / Outcome to be filled.
