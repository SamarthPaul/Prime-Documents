PR-SK-002

Capacity Building

Skilling module of the Diagnostic Framework: records every Past Training the entrepreneur has attended, evaluates its retention / application, and — where a skill has not stuck — flags it for re-training; standalone new skill gaps are captured as New Required Trainings. Soft vs Technical skill areas are tracked separately, each intervention carries a Gap Severity and a six-dimension star-rated Post-Training Assessment scorecard, and a read-only Summary rolls up trainings, gaps and intervention flags.
Key behaviour — the Past → Required re-training bridge. A training is recorded ONCE in the Past Trainings tab. Toggling its “Re-training needed for this skill?” flag captures no cost — it only raises a flag and writes a justification. The flagged training then auto-appears in Required Trainings → Section I (Re-training Interventions), pre-filled (read-only) with the Past Training name, Skill Area and the justification text, where the Fessociate adds Gap Severity, current/required levels, intervention type and an implementation plan. Standalone skill gaps not linked to any past record are added separately in Section II (New Required Trainings). Untoggling the flag (or removing the bridged row) removes it from Section I. Both intervention types share an identical six-dimension Post-Training Assessment scorecard.

On-Screen UI Structure

Top bar: teal header “Capacity Building Module” with crumb “PRIME Rural · DF Capacity Building”, plus an entrepreneur chip (avatar + name) auto-fetched from the profile.
Tab rail: horizontal pill bar with six tabs — Summary · Past Trainings · Required Trainings · Tasks · Resources · Log Book; active tab renders teal-on-white.
Body layout: each tab is a panel. Summary is read-only: a Training-wise Overview table (one row per training), four Aggregate cards (Past Trainings / Required Trainings / Capacity Picture / Status) and a four-cell Intervention Flag strip. Past Trainings renders each training as a collapsible “training block” (sea-blue header with name input + skill-type badge + Remove) holding three sub-sections (Training Details, Skill Retention & Application, Remarks) and a terracotta Intervention Need flag panel. Required Trainings has two sections of repeating rows — Section I Re-training (auto-populated from flagged Past Trainings) and Section II New Required Trainings — each row carrying Skill Type / Skill Area / Gap Severity / levels / intervention + an Implementation Status block + the Post-Training scorecard.
Special widgets: (1) Skill-area multi-select — a grouped listbox split into “— Soft Skills —” and “— Technical Skills —” optgroups that filters to show only the chosen Skill Type’s options. (2) Re-training flag toggle with a live note (“Not flagged” → “↻ Flagged → Details in Required Tab”) and a justification box that bridges to Section I. (3) Six-dimension Post-Training Assessment scorecard — six 1–5 star ratings (Skill / Confidence / Production Improvement, Working Independently, Ability to Train Others, Further Support Needed) over a 1–5 legend (None → Excellent), plus six narrative text fields. (4) Total bars at the foot of Past and Required summarising counts and the Auto-fetch target.
Per Rule 1, every calculated / rolled-up field (all Summary table cells, the four Aggregate cards, the Intervention Flag counts, and both Total bars) renders italic on a cream-deep background and is non-editable, with a tooltip stating its source.

Key validations (whole form): On every Past Training block, Skill Type, Skill Area and Completion Status are mandatory (Skill Area requires at least one selection); End Date must not precede Start Date. The Skill Area listbox must match the chosen Skill Type — selecting Soft Skill clears any technical selections and vice-versa. Toggling “Re-training needed?” On should be accompanied by a Justification and creates exactly one bridged Section I row; turning it Off (or removing that row) clears the flag — the two stay consistent. In Required rows, Required Level should not fall below Current Level. Status / Gap Severity drive every Summary count (Pending = unset status; Critical + High = the red Critical-Gap flag; Completed = the green Interventions-Completed flag). Resources require at least one row. All Summary cells, the four Aggregate cards, the Intervention Flag strip and both Total bars are read-only roll-ups — never edited directly.
