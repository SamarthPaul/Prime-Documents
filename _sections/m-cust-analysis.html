PR-MKT-002

Customer Analysis

Market module framework where the field facilitator maps each product's current customer base across demographic, geographic, psychographic and behavioural segmentation, defines the existing target market, flags products needing an upgrade, and captures an aspirational (Required) target market with an implementation pipeline.

On-Screen UI Structure

Top bar: teal header reading “Customer Analysis” with breadcrumb “PRIME Rural · Marketing Framework”. Below it an entrepreneur chip showing initials avatar and “Auto-fetched from profile” (the entrepreneur is auto-linked, not chosen on this screen).
Tab rail (5 tabs): Summary · Existing · Required · Resources · Task Log. The active tab is solid teal; Existing is the default working tab; Summary opens on load.
Body — Existing & Required tabs: a stacked list of collapsible product blocks. Each product block (sea-green header with product dropdown + status chip + Remove) expands into six numbered sub-sections (I–VI). Sub-sections I–V use a multi-select chip picker (pick one or more values from a master, each shown as a removable chip). Sub-section VI uses single-select dropdowns plus a free-text persona. The Existing block ends with an Intervention Need toggle; the Required block ends with an Implementation Status block.
Special widgets: chip multi-picker (demographic/geographic/psychographic/behavioural/segmentation fields); toggle switch (intervention flag) that reveals a justification box and lights an “Intervention Flagged” chip in the product header; status chip (Pending / In Progress / Completed / Not Feasible) on Required products.
Summary tab is fully calculated — product-wise Existing-vs-Required comparison table, four aggregate cards, and four intervention-flag tiles, all auto-populated from the Existing and Required tabs. Calculated fields are shown in italic on cream-deep background with a tooltip explaining the source (Rule 1). Resources holds a reference-material table; Task Log holds a dated activity log against a fixed task master.

Tab 2 — Required (aspirational target market + implementation)
Caption on screen: “Define the aspirational target market for each product. Track Implementation Status as interventions move through the pipeline.” A Required product block uses the same product header and the same six sub-sections (I–VI) as Existing, but the closing block is Implementation Status instead of Intervention Need. Required rows are matched to Existing rows by product name in the Summary comparison; unmatched Required products appear as “+ New Target Only”.

Segmentation I–VI (aspirational)

Sub-sections I–VI are identical in structure to the Existing tab (Demographic, Geographic, Psychographic, Behavioural, Customer Segmentation, Target Market Definition), but here they describe the aspirational future customer rather than the current base. The same masters and validations apply. See the Existing tab tables above for field-level detail.

Cross-module links: tasks 16 and 17 connect this analysis to the Market Research and Market Linkage modules respectively, keeping the customer definition consistent across the Market framework.
Key validations (whole form): (1) Each product block requires a Product selection before it counts toward the Summary — “Select a product.” (2) Multi-select chip pickers reject duplicate values within the same field. (3) When the Existing Intervention toggle is ON, a Justification is expected and a matching aspirational profile should exist on the Required tab. (4) Income brackets follow Indian number formatting (₹1,00,000 not ₹100,000). (5) The Summary tab and all header status/flag chips are calculated — never edited directly; they recompute whenever Existing/Required data changes. (6) Required products are matched to Existing by exact product name; an unmatched Required product is surfaced as “+ New Target Only” rather than silently dropped.
