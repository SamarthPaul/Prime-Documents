PR-MKT-003

Market Linkage

Part of the Market module, the Market Linkage framework captures where an entrepreneur sells today (Existing channels with revenue), the new channels recommended as an intervention (each holding its own nested product-mapping table with an implementation status), and a month-by-month Sales Tracker of actual recorded sales. A Summary tab rolls everything up into a channel-wise overview, aggregate cards and intervention-flag tiles.
Channel-status aggregate rule: Each recommended channel carries a single status chip whose value is computed from the implementation status of its nested product rows — Completed only when every product row is Completed; Not Feasible if any row is Not Feasible; In Progress if any row is In Progress; otherwise Pending (also the value when a channel has no product rows yet). Type and Selling Mode are set once at the channel level and apply to all products inside it.

On-Screen UI Structure

Top bar: forest-teal header "Market Linkage" with a "PRIME Rural · Marketing Framework" crumb, and an entrepreneur chip below it (avatar + name auto-fetched from profile).
Tab rail (uppercase pills): Summary · Existing · Required · Sales Tracker · Resources · Task Log. Active tab is teal-filled; opening Summary triggers a roll-up refresh.
Body: The Existing tab is a flat channel grid plus an "Intervention Need" toggle block; the Required tab is a channel repeater where each "+ Add Recommended Channel" creates a collapsible channel card with a nested product table; the Sales Tracker is a set of twelve collapsible month accordions, each holding a CRM-style inner table of one row per SKU-per-channel.
Special widgets: (1) collapsible channel cards (sea-coloured header) carrying channel-level Name / Type / Mode / Location / Identified-By plus an aggregate status chip; (2) nested product tables inside each channel with a per-row Implementation Status select; (3) a toggle switch for "Upgrade / Add New Channels?" that reveals a Justification box; (4) source tags (Existing / Recommended / Self-initiated); (5) twelve month accordions each with a header showing the month's auto Qty + Revenue totals; (6) a Summary tab with a channel-wise overview table, four aggregate cards and four intervention-flag tiles.
Calculated fields (Avg Revenue/Month, Annual Revenue, Annual Total Revenue, per-row Tracker Revenue, all month totals, the channel status chip, and every Summary aggregate) render italic on cream-deep background, read-only, with a tooltip per Rule 1.

Tab 3 — Sales Tracker (monthly accordions, actual recorded sales)

Twelve collapsible month accordions (January … December). Each accordion header shows that month's auto Qty and Revenue totals; the section header shows grand Total Units and Annual Revenue chips. Caption: "Each month is a collapsible section. Each row = one SKU via one channel. Price can differ per month. Revenue auto-calculates." "+ Add Row" appends a row inside the open month.

Cohort-year span (Rule 11): the tracker is read against the entrepreneur's cohort / fellowship year — the twelve month buckets belong to the active cohort year, and historical buckets remain visible across the cohort span rather than resetting on calendar year-end. Tracker totals represent actual recorded sales and are kept distinct from the Existing-tab projected revenue in the Summary.

Key validations (whole form): (1) A channel row/card counts only when Channel Name is filled — "Enter the channel name." (2) The recommended channel status chip is system-calculated and read-only: Completed only when all nested product rows are Completed, Not Feasible if any row is Not Feasible, In Progress if any row is In Progress, else Pending — it must never be hand-edited. (3) All numeric fields (Price, Avg Revenue/Month units, Quantity) must be ≥ 0; negatives rejected ("Price cannot be negative"). (4) Existing revenue columns (Avg Revenue/Month, Annual Revenue, Annual Total Revenue) and every Tracker Revenue / month total / grand total are auto-calculated and non-editable. (5) Justification is only valid / shown when the "Upgrade / Add New Channels?" toggle is On. (6) Product SKU is enabled only after a Product Name is chosen (cascading). (7) Tracker month buckets belong to the active cohort year (Rule 11) and Tracker revenue (actual) is kept distinct from projected Existing-channel revenue. (8) Removing a channel deletes all its products (confirm prompt). (9) Resources table must retain at least one row. (10) Task Log entries require a Task Name. (11) All Indian currency uses ₹ with Indian number formatting (e.g., ₹ 1,00,000).
