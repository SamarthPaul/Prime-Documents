PR-MT-002

Labels

Marketing Tools framework that captures the full label-design lifecycle for an entrepreneur's product — existing labels, master & variant-level label content, designer drafts and entrepreneur approval. Variants are not auto-prepopulated: each variant content / status row is created by selecting a variant from a dropdown sourced from the entrepreneur's Product Profile.
Key behaviour. The Variant Content and Label Status child tables are populated by choosing a variant from a dropdown fed by the Product Profile variant list — rows are not pre-created. A Wine / Alcohol Product? toggle on Master Content reveals the statutory alcohol warning block and an ABV field. Storyline text can be drafted by hand or generated by the ✨ Generate with AI action. The Summary tab is fully read-only and rolls up from the Existing and Required tabs.

On-Screen UI Structure

A teal title bar reads Labels Module with crumb PRIME Rural · Marketing Tools. Below sits a horizontal tab rail — Summary · Existing · Required · Resources · Task Log — the active tab highlighted teal.

Summary is read-only: a Variant-wise Overview table (one row per variant, the two right-most columns — Implementation Status & Approved — shaded terracotta / green), four aggregate cards (Existing / Cost / Interventions / Status) and a four-tile Intervention Flags strip.

Existing opens with an auto-fetched Entrepreneur chip, then an Existing Label child table whose Need New Label? column is a toggle; each row carries an Upload / View existing-file widget and a justification field.

Required holds two banded sections. The teal Label Content band wraps Master Content (product-wide fields including the Wine / Alcohol Product? toggle that reveals the statutory warning + ABV) and a nested Variant Content child table whose first column is a variant-selector dropdown (from Product Profile); the table also computes Volume from package L×B×H. The terracotta Label Status band carries a Designer dropdown, a Label Type dropdown and the Label Status child table (variant dropdown + Draft 1/2/3 upload & status cells + Approved-by-Entrepreneur checkbox).

Resources is a reference/asset placeholder. Task Log is a numbered activity table driven by a fixed 14-item task picklist.

Calculated fields (e.g. Volume from package dimensions) render italic on cream-deep background, are never editable and carry a formula tooltip (Rule 1).

Key validations (whole form): Product Name is required on Master Content. Variant Name must be selected (from the Product Profile dropdown) on every Variant Content and Label Status row — rows are never auto-created. When Wine / Alcohol Product? is ON, the statutory warning banner is shown and Alcohol Content (ABV) is required. Each child table must retain at least one row (“At least one row is required” on delete of the last row). File uploads accept image / PDF (artwork) or image-only (photos) as noted. Date fields use DD/MM/YY; Best Before should fall after MFD. Numeric dimensions, MRP and weights cannot be negative; Volume is auto-calculated (L×B×H) and read-only. Variants are matched across the Existing, Variant Content and Label Status tables by name to assemble the Summary roll-up.
